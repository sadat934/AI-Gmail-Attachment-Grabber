Privacy Policy for GrabMail

Effective Date: June 7, 2026
Last Updated: June 7, 2026

Introduction

GrabMail ("we", "our", "the Extension") is a Chrome extension that helps users download email attachments from Gmail. This Privacy Policy explains our data practices in compliance with Chrome Web Store policies, GDPR, CCPA, and other privacy regulations.

Our Core Privacy Principle: GrabMail does NOT collect, store, transmit, or share any personal user data beyond what is absolutely necessary to perform its stated function, and all such data is processed temporarily and discarded immediately.

What Data We Do NOT Collect

We explicitly do NOT collect:

  • Personal identifiable information (names, addresses, emails, phone numbers)
  • Financial or payment information
  • Authentication passwords or long-term credentials
  • Web browsing activity or history
  • Email body content or personal communications
  • Location data or IP addresses
  • Usage analytics or telemetry
  • Any data for sale to third parties
  • Any data for advertising or marketing purposes

What Data We Access Temporarily

To perform its function (downloading Gmail attachments), GrabMail temporarily accesses:

Gmail Attachment Metadata (Not Stored)

What: Attachment filename, file size, MIME type, email sender, email subject, email date
Why: To identify which attachments match your search criteria
How Long: Processed in memory only, discarded immediately after use
Storage: No permanent storage - data exists only during active use

Your Preferences (Stored Locally Only)

What: AI provider choice, API key (if using AI), last search settings, download preferences
Why: To remember your preferences between sessions
Where: Chrome's local storage (chrome.storage.local) on your device only
Control: You can clear anytime by signing out or uninstalling

How We Use Data

  1. Search Function: You enter search criteria → Extension queries Gmail API → Gmail returns matching email IDs → Extension fetches metadata → Results displayed → Data discarded
  2. Download Function: You select files → Extension downloads attachments from Gmail → Files saved to your Downloads folder → No intermediate storage → No copies retained
  3. AI Mode (Optional): If you use AI mode, only filename, sender, subject, and file size are sent to your chosen AI provider. Your API key goes directly to the AI provider. You can avoid this entirely by using Manual Search mode

Permissions Explained

  • identity - Gmail OAuth sign-in (no password handling)
  • downloads - Save files to Downloads folder
  • storage - Remember preferences locally
  • gmail.googleapis.com - Access Gmail API (read-only)

Third-Party Services

Gmail API (Required)

  • Provider: Google
  • Purpose: To access email metadata and download attachments
  • Data Shared: OAuth token, email queries, email/attachment IDs
  • Privacy Policy: https://policies.google.com/privacy

AI Providers (Optional)

Only if you use AI Search mode:

  • Claude (Anthropic): https://www.anthropic.com/privacy
  • Gemini (Google): https://ai.google.dev/terms
  • OpenRouter: https://openrouter.ai/privacy

Data Sent: Filename, sender, subject, file size only
Opt-Out: Use Manual Search mode

Data Retention

We retain ZERO data.

  • No servers, no databases, no logs
  • When you uninstall: All local preferences deleted by Chrome
  • Your Gmail data remains untouched
  • Downloaded files remain in your Downloads folder

Your Rights

  • Access: Review our open-source code
  • Delete: Uninstall extension anytime
  • Opt-Out: Don't use AI mode if you prefer
  • Revoke: Remove Gmail access at https://myaccount.google.com/permissions

To remove all data: Sign out → Uninstall extension → (Optional) Revoke OAuth at Google Account

Children's Privacy

We don't collect data from anyone, including children. COPPA and GDPR compliant by design (no data collection = no age restrictions needed). Parental supervision recommended for minors.
